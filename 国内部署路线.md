# 国内部署路线（别人不翻墙也能访问）

选股网站需要「页面」+「接口代理」才能完整可用。下面是一条**国内可访问**的部署路线。

---

## 路线总览

| 步骤 | 做什么 | 得到什么 |
|------|--------|----------|
| 1 | 腾讯云 云开发 CloudBase 开通环境 | 一个环境 ID、默认域名 |
| 2 | 上传前端静态包（dist）到「静态网站托管」 | 页面国内可访问 |
| 3 | 部署一个「API 代理」云函数，并绑定到 /api | 接口国内可访问 |
| 4 | （可选）绑定自己的域名 | 用自己域名访问 |

完成后，别人用你给的链接（如 `https://xxx.tcloudbaseapp.com`）即可访问，**无需注册、无需翻墙**。

---

## 第一步：开通腾讯云云开发

1. 打开 **https://cloud.tencent.com/product/tcb**，点击「立即使用」。
2. 用微信或 QQ 登录腾讯云（新用户需简单实名）。
3. 进入 **云开发控制台** → **新建环境**（选「按量付费」即可，静态+云函数免费额度够用）。
4. 记下 **环境 ID**（如 `stock-xxx`）和 **默认域名**（如 `stock-xxx-xxx.tcloudbaseapp.com`）。

---

## 第二步：部署前端静态包

1. 在本地「选股网站」目录执行：
   ```bat
   npm run build
   ```
2. 在云开发控制台 → 你的环境 → **静态网站托管** → **开通**。
3. 开通后进入「静态网站托管」→ **上传文件**，把本地的 **`dist` 目录里的全部内容**（不要传 dist 文件夹本身）上传到根目录。
   - 即：`dist/index.html`、`dist/assets/` 等都在托管根目录。
4. 在「静态网站托管」里打开「设置」→ 把「默认首页」设为 `index.html`（一般已是）。
5. 记下静态托管的访问地址（如 `https://xxx.tcloudbaseapp.com`），先浏览器打开看是否能看到页面（此时点「开始筛选」会报错，因为还没配接口，正常）。

---

## 第三步：部署 API 代理云函数

选股网站会请求 `/api/em`、`/api/emhis`、`/api/emdc`、`/api/sina`，需要有一个「云函数」在国内帮你转发到东方财富/新浪。

1. 在云开发控制台 → **云函数** → **新建云函数**：
   - 函数名称：`apiproxy`
   - 运行环境：**Node.js 16**
   - 提交方式：本地上传 zip（见下）

2. 在本地新建一个文件夹，例如 `apiproxy`，在里面建两个文件：

   **package.json**（云函数依赖，可为空）：
   ```json
   {
     "name": "apiproxy",
     "version": "1.0.0",
     "main": "index.js"
   }
   ```

   **index.js**（代理逻辑）：
   ```javascript
   const https = require('https');
   const UA = 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/120.0.0.0';

   function get(url) {
     return new Promise((resolve, reject) => {
       https.get(url, { headers: { 'User-Agent': UA } }, (res) => {
         let data = '';
         res.on('data', chunk => data += chunk);
         res.on('end', () => resolve({ status: res.statusCode, data }));
       }).on('error', reject);
     });
   }

   const HOSTS = {
     '/api/em': 'push2.eastmoney.com',
     '/api/emhis': 'push2his.eastmoney.com',
     '/api/emdc': 'datacenter-web.eastmoney.com',
     '/api/sina': 'money.finance.sina.com.cn',
   };

   function getHost(path) {
     for (const p in HOSTS) if (path.startsWith(p)) return { prefix: p, host: HOSTS[p] };
     return null;
   }

   exports.main = async (event, context) => {
     const path = (event.path || event.requestContext?.path || '').replace(/^\//, '');
     const q = event.queryStringParameters || {};
     const qs = new URLSearchParams(q).toString();
     const target = getHost('/' + path);
     if (!target) return { statusCode: 404, body: 'Not Found' };
     const rest = path.replace(target.prefix.replace(/^\//, ''), '').replace(/^\//, '');
     const url = `https://${target.host}/${rest}${qs ? '?' + qs : ''}`;
     try {
       const r = await get(url);
       return { statusCode: r.status, body: r.data, headers: { 'Content-Type': 'application/json' } };
     } catch (e) {
       return { statusCode: 502, body: String(e.message) };
     }
   };
   ```

3. 把 `apiproxy` 文件夹打成 **zip**（zip 内要包含 `index.js` 和 `package.json`），在云函数里选择「本地上传」上传该 zip，保存并部署。

4. **绑定 HTTP 触发**：
   - 在云函数 `apiproxy` 详情里 → **触发器** → **创建触发器**。
   - 类型选 **HTTP 触发**，路径填 `/api` 或 `/api/*`（按控制台提示，不同地域可能写 `apiproxy` 或路径前缀）。
   - 若控制台是「网关 + 路径映射」：在 **API 网关/HTTP 服务** 里添加「路径 `/api/*` → 云函数 `apiproxy`」，并发布。

5. 确认云开发的「HTTP 访问服务」或「自定义域名」里，你的环境域名下 `/api/*` 会走到该云函数（具体名称以腾讯云当前控制台为准）。

---

## 第四步：让前端请求走云函数的 /api

当前前端写的是相对路径 `/api/em` 等，部署在云开发静态托管后，同一域名下 `/api/*` 必须由上面那个云函数处理。

- **若你用的「云开发 Web 应用」/「静态托管 + 同一域名下云函数」**：  
  静态托管和 HTTP 触发在同一域名（如 `https://xxx.tcloudbaseapp.com`）下，则 **无需改前端**：页面在 `https://xxx.tcloudbaseapp.com`，请求 `/api/em` 会发到同域，由网关转到云函数即可。

- **若静态托管与云函数域名不一致**：  
  需要在前端把接口 base 改成云函数所在域名（例如通过环境变量 `VITE_API_BASE`），再重新 build 并上传静态包。本仓库默认同域部署，一般不用改。

---

## 第五步：验证与分享链接

1. 打开你的静态托管地址（如 `https://xxx.tcloudbaseapp.com`）。
2. 点「开始筛选」或「数据诊断」，能正常拉数据即表示接口代理生效。
3. 把该链接发给别人，**对方无需注册腾讯云、无需 Vercel、无需翻墙**，直接打开即可使用。

---

## 可选：绑定自己的域名

若你有已备案的域名（如 `tongyu.com`）：

1. 在云开发控制台 → **静态网站托管** / **HTTP 服务** → **自定义域名**。
2. 添加域名，按提示在域名服务商处配置 CNAME 或 A 记录。
3. 备案域名在国内解析后，即可用 `https://你的域名` 访问。

---

## 简要对照

- **Vercel**：国外，国内常被墙，别人不翻墙经常打不开。
- **本路线（腾讯云云开发）**：静态 + 云函数都在国内，别人直接打开链接即可访问，无需注册、无需翻墙。

如某一步控制台界面与上述不一致，可按「腾讯云云开发 静态托管」「云函数 HTTP 触发」在腾讯云文档里查当前步骤。
